# Secure EMR File Access Simulation

## Overview
This Python project simulates a secure access system for electronic medical records (EMR). It enforces access attempts, validates file paths, and ensures safe handling of sensitive patient information.

## Objectives
- Simulate secure access to patient EMR files
- Enforce maximum login attempts
- Handle file not found, permission, and system errors
- Demonstrate basic EMR security and patient data protection

## Features
- Access attempt limitation
- Error handling for invalid paths and permissions
- Secure file handling with automatic closure
- Console-based EMR viewing simulation

## Technologies
- Python 3

## Healthcare Relevance
Protecting patient data is critical in healthcare. This system demonstrates access control, file security, and error handling in EMR systems, reflecting key health informatics and digital health compliance concepts.

## Future Improvements
- Integrate with authentication (user login, roles)
- Store EMR in encrypted SQLite or database
- Log access attempts with timestamps
- Add GUI for secure EMR portal simulation
